# Revolver Rift — Backend Architecture

## Overview

The backend is a **Strapi v5.36.1** headless CMS application running on **Node.js ≥20**, connected to a **PostgreSQL** database (`revolverriftcms`). It exposes a REST API consumed by the React/Vite frontend.

- **Admin panel**: `http://localhost:1337/admin`
- **API base URL**: `http://localhost:1337/api`
- **Port**: `1337` (configurable via `.env`)

---

| Layer | Tech |

## Reusable Components

Components are NOT standalone API endpoints — they are field groups embedded inside content types.

### `sections` (page-level blocks)

| Component | Used In | Fields |
|-----------|---------|--------|
| `sections.hero-section` | `home-page` (1×) | `title` (string), `subtitle` (text), `background_image` (media), `button_text` (string), `button_link` (string) |
| `sections.features-section` | `home-page` (1×) | `section_title` (string), `features` → repeatable `elements.feature-item` |
| `sections.about-section` | `home-page` (repeatable) | `title` (string), `description` (blocks/rich-text), `statistics` → repeatable `elements.statistic-item` |
| `sections.trailer-section` | `home-page` (repeatable) | `thumbnail` (media), `video_url` (string) |

### `elements` (atomic field groups)

| Component | Used In | Fields |
|-----------|---------|--------|
| `elements.feature-item` | `features-section` | `title` (string), `description` (text), `icon` (media) |
| `elements.statistic-item` | `about-section` | `number` (integer), `label` (string) |

## API Configuration

```js
// config/api.js
rest: {
  defaultLimit: 25,
  maxLimit: 100,
  withCount: true,
}
```

Standard REST endpoints auto-generated by Strapi:
- `GET /api/blogs` → list all blog posts
- `GET /api/blogs/:id` → single blog post
- `GET /api/home-page` → the single home-page record (with `?populate=*` to get components)
- etc.

## Environment Variables (`.env`)

| Variable | Purpose |
|----------|---------|
| `HOST` | Server host (default `0.0.0.0`) |
| `PORT` | Server port (default `1337`) |
| `APP_KEYS` | 4× base64 keys for session cookies (comma-separated) |
| `API_TOKEN_SALT` | Hashes API tokens stored in DB — must match across shared DB setups |
| `ADMIN_JWT_SECRET` | Signs admin panel JWT sessions |
| `TRANSFER_TOKEN_SALT` | Hashes data transfer tokens |
| `ENCRYPTION_KEY` | Encrypts sensitive stored data |
| `JWT_SECRET` | Signs user auth tokens |
| `DATABASE_CLIENT` | `postgres` |
| `DATABASE_HOST` | `localhost` |
| `DATABASE_PORT` | `5432` |
| `DATABASE_NAME` | `revolverriftcms` |
| `DATABASE_USERNAME` | `postgres` |
| `DATABASE_PASSWORD` | *(personal, not committed)* |
| `DATABASE_SSL` | `false` for local, `true` for cloud (Neon/Supabase) |
| `DATABASE_URL` | Alternative: full connection string (used for cloud DBs) |
| `ADMIN_URL` / `SERVER_URL` | Optional ngrok tunnel URLs for external access |

## Key Observations & Gaps

1. **Blog is the only fully implemented API** — all other content types (`product`, `content`, `showcase-gallery`, `developer-quote`, `partner-stage`, `showcase-page`, `news-page`, `partners-page`, `credits-page`) have empty schemas with no fields defined.

2. **Home Page is structured but data-driven** — uses a component-based architecture (hero → features → about → trailer), meaning all homepage copy and media is managed from Strapi admin, not hardcoded in React.

3. **No custom controllers, services, or routes** — all 11 APIs use Strapi's auto-generated CRUD. No custom business logic in backend code.

4. **No custom migrations** — `database/migrations/` is empty. Strapi auto-manages schema via its own internal migration system.

5. **No custom plugins configured** — `config/plugins.js` is empty.

6. **Middleware is default Strapi stack** — no custom CORS origins set (potential issue if frontend runs on a port other than what Strapi expects).

7. **`register()` and `bootstrap()` hooks are empty** in `src/index.js` — no seed data, no lifecycle hooks.
