Revolver Rift - Strapi Content Management System Manual

Welcome to the Revolver Rift Strapi CMS User Manual. This guide will step you through managing your backend content, customizing your website text and images, and managing users and permissions within the Strapi Admin Panel.

1. Accessing the Admin Panel

To begin managing content, you need to log in to your Strapi Administration Panel.

  1. URL: Navigate to your Strapi login page at: https://revolverrift-production.up.railway.app/admin/auth/login. The base URL for the admin dashboard is https://revolverrift-production.up.railway.app.
  2. Login Credentials: Enter your assigned Email and Password.
  3. Dashboard: Upon successful login, you will land on the Strapi Dashboard, your central hub for managing the Revolver Rift content.

2. Navigating the Interface

On the left-hand sidebar, you will see a navigation menu categorized into a few key sections:

  • Content Manager: Where you spend most of your time. This is where you Add, Edit, and Delete content.
  • Content-Type Builder: (For Developers Only) This is where the structure (Schema) of your content is built. You generally will not need to touch this.
  • Media Library: A global repository for all your uploaded images, videos, and files.
  • Settings: Where you manage administrative configurations, users, roles, and API permissions.
💡 TIP Use the Content Manager to input data, and the Media Library to manage your files before linking them to content!

3. Understanding Content Types

Strapi organizes your site's data into two distinct categories: Collection Types and Single Types.

📖 Collection Types

These are items that you can have many of (like a list). When you click a Collection Type, you'll see a list of entries.

  • Examples in Revolver Rift: News Items, Products, Gallery Items, Blogs, Developer Quotes, Product Categories.

📄 Single Types

These represent a single unique page or structural component strictly meant to have just one instance.

  • Examples in Revolver Rift: Home Page, News Page, Contact Page, Shop Page, Partners Page, Credits Page, Showcase Page.

A. The Home Page (Single Type)

This controls the main landing page components. It is structured into dynamic zones/components.

  • Hero: The top banner (Title, Subtitles, Background Image/Video).
  • Features: Section highlighting game/product features.
  • About: Section providing lore or project descriptions.
  • Gallery Items: A list mapping directly to the Gallery Item Collection Type.
  • Trailer: Link and thumbnail for the primary cinematic or gameplay trailer.

B. News Page (Single Type)

Controls the top-level layout for the news/blog listing page.

  • Page Title: Usually "ALL DEVELOPER BLOGS".
  • Subtitle: Catchy one-liner (e.g., "// Official Developer Logs").
  • Background Texture: Image applied to the page background.
  • SEO Title & Description: Used for Google search indexing.

C. News Items (Collection Type)

Individual blog or news entries. Needs to be published to show up on the News Page.

  • Title: Headline of the news.
  • Badge Label & Color: e.g., "Feature Analysis", with choices like Red or Beige.
  • Excerpt: Short summary for the list view.
  • Content: The rich-text or blocks builder for the main article itself.
  • Cover Image: The main thumbnail image.
  • Publish Date & Order: Sorting configurations.

D. Products (Collection Type)

Products tied to the shop ecosystem.

  • Name & Slug: Product title and its unique URL identifier.
  • Tagline: Short subtext (e.g., "Limited Edition Hoodie").
  • Description: The full details.
  • Price & Compare-at Price: Define selling price and the original price (for discounts).
  • SKU: Inventory code tracking.
  • Product Type: (Apparel, Digital, Weapon Skin, Collectible).
  • Status: (Available, Sold Out, Coming Soon, Archived).
  • Media: Thumbnail, Gallery (multiple images), and Hover Image (swap on hover).

E. Gallery Items (Collection Type)

Individual visual assets showcased on the Home or Showcase pages.

  • Image: An image, video, or audio file.
  • Title & Description: Context text.
  • Link & Button Text: Call to action link for the gallery item.

5. Adding and Editing Content (The Daily Workflow)

Creating New Content

  1. Navigate to Content Manager -> Select a Collection Type (e.g., News Items).
  2. Click the vast, blue + Create new entry button in the top right.
  3. Fill in the required fields (often marked with a star *).

Editing Text & Managing Blocks

  • Standard text fields can be typed directly.
  • Blocks/Rich Text: Allow you to format text, inject links, make bold/italic, and add inner-headers.

Adding and Changing Images

  1. Click the + Add or click an existing image area on fields like cover_image or thumbnail.
  2. A modal will pop up: You can either upload a new file from your computer or pick an existing one from the Media Library.
  3. To change an image, hover over the current image, click the Trash icon to remove it, and then upload a new one.

🔄 The Save & Publish Workflow

⚠️ IMPORTANT If a document is set to "Draft", it will not appear on your website frontend. You must click Publish for the public to see it. If you edit a currently Published document, clicking Save immediately pushes those changes live.

6. Media Library Management

The Media Library tab on the left allows bulk management of assets.

  • You can upload multiple files at once.
  • You can create Folders to organize assets (e.g., "Shop Images", "Hero Banners", "Blog Covers").
  • This makes searching for an image while creating a Product or News Item incredibly fast.

7. Settings: Users, Roles, and API Permissions

Occasionally, you need to add a new employee/admin or ensure your Frontend website is allowed to read specific APIs.

Managing Admin Users & Roles (Who can log in)

  1. Go to Settings -> Administration Panel -> Roles.
  2. Super Admin: Has access to everything.
  3. Editor/Author: Standard roles that can manage content but cannot change settings.
  4. Go to Settings -> Administration Panel -> Users to invite new people. Click + Invite new user, assign them a Role, and an invitation will be sent.

Managing API Permissions (How your site pulls data)

If a new Page or Content Type isn't showing up on your website even though it's "Published", it is likely due to API permissions dropping. Your frontend needs permission to "read" the data.

  1. Go to Settings -> Users & Permissions Plugin -> Roles.
  2. Click on the Public role (This represents your unlogged frontend website visitors).
  3. Scroll down to the Permissions central matrix.
  4. Locate the specific Content Type (e.g., Product or Home-Page).
  5. Check the following boxes:
    • find: Allows the frontend to fetch a list of the items.
    • findOne: Allows the frontend to fetch the specific single item.
  6. Click Save in the top right corner. The website can now retrieve the data!
🛑 CAUTION For the Public role, almost never check create, update, delete, or destroy unless you explicitly want strangers on the internet deleting your databases. Stick to find and findOne.

Conclusion

You are now fully equipped to jump in and manipulate the Revolver Rift content. Whenever you're in doubt:

  1. Ensure it's Published.
  2. Ensure the find/findOne API permissions are activated for Public.
  3. Don't forget that Changes require a 'Save'!
